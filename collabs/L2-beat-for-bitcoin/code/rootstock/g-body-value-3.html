L1 Verification

None

Bitcoin is not used to verify the state of the Rootstock blockchain. All transactions and state updates are stored and verified on the Rootstock blockchain.

Consensus

Merge-mining

Rootstock is merge-mined. Currently, more than 35% of Bitcoin miners are engaged in RSK merge-mining.

Bitcoin Hashpower

Rootstock transaction finality and block ordering is not secured by Bitcoin hashpower

RSK is a sidechain. This means that Bitcoin hashpower doesn't directly influence the transaction ordering and finality of RSK blocks. RSK has a different difficulty and hash rate than Bitcoin. While Bitcoin miners merge-mine RSK, and there are security benefits to having that architecture, miners of the RSK network are still considered "RSK miners".

Alternative Token

None

Rootstock does not have a custom token. The native token on the Rootstock blockchain is RBTC, which is a token backed by BTC.

Technology

State correctness

Rootstock miners (Bitcoin miners who merge-mine the Rootstock blockchain) enforce state correctness.

All transactions are recorded on the RSK blockchain

All transactions that are executed on the RSK blockchain are recorded on the RSK ledger. All RSK nodes can view these transactions.

Data availability

Rootstock stores data on the Rootstock blockchain. Anyone running an RSK node can access and verify the state of the RSK blockchain.

Operator

Open

Block production is permissionless. Anyone who is willing to merge-mine can produce blocks and earn rewards. If interested parties do not possess enough capital to compete in the Bitcoin mining market, they can join a mining pool.

Withdrawals

Reliant on Rootstock signers

Withdrawals are currently permissioned by a federated group of signers. Users must trust, that when they deposit BTC into the Roostock blockchain, the signers will not collude and steal their BTC. Learn more about the Powpeg mutli-sig here.

Source Code

Open-source

Rootstock is open-source. The node software is also open-source, and anyone can run an RSK node.
